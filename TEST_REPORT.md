## 6. 测试风险与建议

### 6.1 已识别风险

| 风险项 | 等级 | 说明 | 缓解措施 |
|--------|------|------|----------|
| SQLite 并发写入 | 中 | 高并发写入可能出现锁冲突 | 当前万级数据量无影响；10万+时建议迁移至 PostgreSQL |
| Access Token 明文存储 | 中 | `repository_configs.access_token` 为明文 | 后续版本升级为 AES 加密存储 |
| 无用户认证 | 低 | 当前无登录/权限控制 | 内网部署使用；公网部署需增加 JWT 认证 |
| 浏览器兼容性 | 低 | 不支持 IE | 目标用户均使用现代浏览器，风险可控 |

### 6.2 后续测试建议

| 建议项 | 优先级 | 说明 |
|--------|--------|------|
| 自动化测试覆盖 | P1 | 使用 pytest + jest 编写单元测试和集成测试 |
| 性能压测 | P2 | 数据量增长至 10 万+ 时进行全链路压测 |
